# コード譜言語仕様書

- バージョン: **1.0**（正式版、2026-07-04 確定）
- 開発中の暫定版番号（v1.1〜v1.7.x）の履歴は git ログおよび [spec-test-matrix.md](./spec-test-matrix.md) を参照

## 概要

数字ディグリー記譜法（ナッシュビル・ナンバー・システム系）に基づくコード譜の DSL。Markdown 文書に埋め込み、ブラウザ上で次のことができる:

- ディグリー（ローマ数字）⇄ 実音コード名のタブ切り替え表示と、任意キーへの移調
- コードごとの歌詞・セクションラベル・強調色
- 和音の再生（キー・テンポ・拍子に追従）
- 譜面の PNG 画像コピー

## Markdown への埋め込み

Markdown 文書には **`:::` 単独のフェンス**でコード譜ブロックを埋め込む:

```
:::
---
key: G
---
1 3m7 4M7 5 | b7 1
:::
```

- 開き・閉じとも `:::`（コロン 3 つ以上）の行。言語名は書かない（このプラットフォーム専用のコード譜ブロック）。閉じフェンスのコロン数は**開き以上**が必要（CommonMark のフェンス規則に準拠。`::::` を `:::` では閉じられない）
- ` ``` ` フェンスは言語タグに関わらず通常のコードブロックであり、コード譜にはならない
- `:::` 直後の文字列は無視される（将来 `:::名前` 形式を別種のコンテナに使う可能性を予約）

## フロントマター

ブロックの**先頭にのみ**、YAML 風のフロントマターを置ける:

```
フロントマター = "---" 改行 フィールド行* "---" 改行
フィールド行   = フィールド名 ":" 空白* 値
```

- フロントマターは**省略可能**。置く場合は 1 行目が `---` で、閉じの `---` 行が**必須**。閉じが無い場合は UnclosedFrontmatter エラーとなり、**以降の全行がフロントマター内と見なされるため本文はパースされない**（「正しい行の継続」の唯一の例外）
- コロンの後の空白は省略できる（`key:G` も可）

| フィールド | 値 | 既定値 | 説明 |
|-----------|-----|--------|------|
| `key` | `A`〜`G` + 任意の `#`/`b`（例: `C` `G` `F#` `Bb`）。メジャーキーのみ | `C` | コードタブ・再生の初期キー。AST 上は宣言の有無を保持し、既定値の適用は表示層が行う |
| `bpm` | 整数 20-400（4 分音符基準） | `120` | 再生テンポ |
| `time` | `1/2` `2/2` `2/4` `3/4` `4/4` `5/4` `6/4` `3/8` `6/8` `9/8` `12/8`（ホワイトリスト制） | `4/4` | 拍子。再生時のスロット長とベースの刻みに反映 |

- 未知のフィールドは InvalidFrontmatterField、不正な値は InvalidKey / InvalidTempo / InvalidTimeSignature エラー

## 基本構造

```
入力     = [フロントマター] 行*
行       = コード行 | セクションラベル行 | 歌詞行
コード行 = (コード | グループ | 小節線 | 反復 | NC | 空拍)* [コメント]
セクションラベル行 = "[" 名前 "]" [コメント]
歌詞行   = ">" 断片*
グループ = コード ("-" コード)+
反復     = "%"
NC       = "NC" | "N.C."
空拍     = "_"
コメント = "#" 任意のテキスト（行末まで）
```

### パースの基本方針

- **fail-fast はしない**。エラーのある行は結果から除外してエラーとして収集し、正しい行のパースは継続する（部分 AST + エラー列を返す）。ライブプレビューで 1 文字ごとの不完全入力に耐えるための設計
- エラーには**行番号**（元ソース基準、1-indexed）が付く。コード行由来のエラーには **position**（行内の 0-indexed 文字位置）も付く（フロントマター・歌詞行のエラーは行単位のため position なし）
- **空入力・空白のみの入力**は、行を持たない空の楽譜として正常に扱う（エラーではない）
- 空行は無視される（行番号のカウントには含まれる）

## トークンの定義

### 1. コード (Chord)

空白文字（スペース、タブ等）で区切られた文字列で、以下の構造を持つ:

```
コード = [アクシデンタル] ディグリー [クオリティ] [セブンス] [オルタレーション] [括弧内オプション] [スラッシュベース] [色アノテーション]
```

#### 構成要素

- **アクシデンタル**: `s` (シャープ) | `b` (フラット)
- **ディグリー**: `1` | `2` | `3` | `4` | `5` | `6` | `7`
- **クオリティ**: `m` (マイナー) | `dim` (ディミニッシュ) | `aug` (オーギュメント)
- **セブンス**: `7` | `M7` | `6` | `add` + Addディグリー
  - **パース優先順位**: `M7` → `7` → `6` → `add`（`M7` は 2 文字なので先にチェック）
  - **Addディグリー**: `2` | `4` | `6` | `9` | `11` | `13`
  - セブンスの直後に再びセブンスがマッチする場合（`1M77` `17M7` `1M7M7` `1M7add9` 等）は DuplicateSeventh エラー
- **オルタレーション**: `b5` (フラットファイブ) | `sus` + ディグリー (2/4/6 のみ)
  - 括弧外のオルタレーション（括弧内表記と併用可能で、両方 AST に保持される）
- **テンション**: `b9` | `9` | `s9` | `11` | `s11` | `b13` | `13`
- **括弧内オプション**: `(` + オプション + (`,` + オプション)* + `)`
  - **オプション**: テンション | オルタレーション
  - 空の括弧 `()` はエラー、カンマの不正使用（先頭・末尾・連続）はエラー
  - **括弧の内側では空白を許容する**（`1M7(b9, s11)` は 1 つのコードトークン）
- **スラッシュベース**: `/` + [アクシデンタル] + ディグリー
  - スラッシュがある場合、ディグリーは**必須**（欠落は MissingBassDegree。`1/8` のように 1-7 以外の度数は InvalidDegree）
  - アクシデンタルは**オプショナル**
- **色アノテーション**: `@` + 色名（`blue` | `red` | `green`）
  - コードトークンの**末尾**にのみ置ける（スラッシュベースよりも後）
  - ブログ記事中で文脈上目立たせたいコードに使う**表示用アノテーション**（音楽的な意味は持たない）
  - 色名は小文字のみ。`blue` / `red` / `green` 以外はエラー

### 2. グループ（`-` で 1 スロットを共有）

`-` でコードを連結すると、複数のコードが**表示上の 1 スロット**を共有する（四分音符に対する八分音符のような分割のイメージ）:

```
6m7 3m/5 4m 2-5    # 最後のスロットを 2 と 5 が等間隔で分け合う
1-2m7-5@red        # 3 連結。色は個別のコードに付く
```

- 各パートは通常のコードとして完全にパースされる（色アノテーションも個別に付けられる）
- 空のパート（`2-` / `-5` / `2--5`）はエラー。区切り文字だけのトークン（`---` 等）は 1 つの UnrecognizedToken になる
- 括弧の内側では分割しない
- **設計判断**: `-` をグループ区切りに採用したため、ジャズ記法のマイナー略記（`6-` = 6m）は**サポートしない**。マイナーは常に `m` で書く

### 3. 小節線 (BarLine)

`|` は小節区切りを表す。空白なしでもトークンを区切る（`1|4` = `1` `|` `4`）。

### 4. 反復記号 `%`

`%` は**直前のコードの反復**（同じコードが続くこと）を表す省略記号。独立したトークンで、空白なしでも区切られる（`1%` は `1` と `%`）。

```
| 1M7 % | 4 % |     # 1M7 1M7 | 4 4 と同じ意味
```

- パーサは反復の解決（どのコードを指すか）を行わず、AST には `Repeat` トークンとして残る。表示はディグリー・コード両タブとも `%`
- `%` に色アノテーション等は付けられない

### 5. N.C.

`NC` または `N.C.` と書くと「コードなし」を表すトークンになる（表示は常に `N.C.`）。ブレイクやストップタイムに使う。

```
| NC 1 | N.C. 4 |
```

- 1 スロットを占有し、**歌詞を割り当てられる**（ストップタイムで歌う場面）
- 色アノテーション等の注釈は付けられない（`NC@red` はエラー）
- グループ（`-` 連結）の中では使えない

### 6. 空拍 `_`

`_` は**空の拍**を表すトークン。スロットを 1 つ占有するが、コードは表示されない。拍位置を明示したいとき（コードを伸ばす・歌詞だけ置く）に使う。

```
| 1 _ _ _ |          # 1 を 4 スロット分に伸ばすイメージ
> ひ か り の        # 空拍にも歌詞を割り当てられる
```

- 空白なしでも区切られる（`1_4` は `1` `_` `4` の 3 スロット）
- **歌詞の割り当て対象**（`||` 由来の空セルは対象外）
- グループの中では使えない

### 7. コメント

`#` から行末まではコメントとして無視される。行頭に置けば行全体がコメントになる。

```
1 4 5 | 1   # イントロ
# この行はすべてコメント
```

歌詞行は例外で、`#` も歌詞の一部になる（自由テキスト優先）。

## セクションラベル

`[名前]` を**単独行**で書くと、セクションラベルとして扱われる。中身はフリーテキスト（自動採番はしない）。

```
[Aメロ]
| 1M7 % | 4M7 5 |

[サビ]
| 4M7 5 | 3m7 6m7 |
```

- ラベル行にもコメント（`#` 以降）を書ける。そのためラベル名に `#` は使えない
- `[A] 1 4` のように同じ行に他のトークンがある場合はラベルではなく、通常のコード行として解釈される（`[A]` は `[` がディグリーとして不正なため InvalidDegree エラー）
- 表示は**白抜きの反転バッジ**（背景 = 本文の文字色、文字 = ページ背景色。ライト/ダーク両テーマで自動反転）
- AST では `sections: [{line, name}]` として行番号つきで保持される

## 歌詞行

コード行の**直後の行**に `>` で始まる行を書くと、空白区切りの断片が直前のコード行のスロット（コード / グループ / `%` / `N.C.` / 空拍 `_`）へ**左から順に**割り当てられる。表示は各コードの真下。

```
| 1M7 % | 4M7 5 |
> ひかりの _ さきへ ゆこう

| 6m7 3m7 | 2-5 1 |
> きみと あるく この-みち へ
```

- **完全にオプショナル**: 歌詞行が無ければコード行の書き方・表示は一切変わらない
- `_` 単独 = そのスロットには歌詞なし（1 つスキップ）
- 断片中の `_` は表示上のスペースになる（`sweet_home` → `sweet home`。断片を分けずにスペースを含めるための記法）
- 断片がスロット数より**少ない**のは正常（残りは歌詞なし）。**多い**場合は TooManyLyricFragments エラー（先頭から割り当てた上でコード行の描画は維持される）
- グループ（`2-5`）への断片は、`-` でグループのコード数どおりに分割できる場合に各コードへ配られる（`この-みち` → `2` に「この」、`5` に「みち」）。分割できない場合は先頭のコードにまとめて付く
- 歌詞行は**自由テキスト**であり、`#` もコメントにならず歌詞の一部になる
- 歌詞行の直前の行がコード行でない場合（ブロック先頭・空行やラベルの直後・歌詞行の連続・**直前のコード行がパースエラーで除外された場合**）は OrphanLyricLine エラー
- コードタブ（実音表示）・任意キーへの移調でも歌詞は保持される

## 構文例

### 基本形
```
1          # ディグリー1
3m         # ディグリー3マイナー
b7         # フラット7
s4         # シャープ4
```

### セブンス付き
```
5M7        # ディグリー5メジャーセブンス
2m7        # ディグリー2マイナーセブンス
47         # ディグリー4ドミナントセブンス
36         # ディグリー3シックス
4add6      # ディグリー4 add6
4add9      # ディグリー4 add9
3dim7      # ディグリー3ディミニッシュセブンス
```

### オルタレーション（括弧なし・括弧あり両対応）
```
1sus4      # 括弧なし表記
4M7b5
1(sus4)    # 括弧あり表記（推奨）
4M7(b5)
```

### テンション付き
```
1M7(b9)
2m7(b9,s11)
1M7(b9,s11,b13)
1M7(b9, s11)      # 括弧内の空白は許容
```

### 混在・ベース音・色
```
1M7b5(b9)             # 括弧外 b5 + 括弧内 b9（両方 AST に保持）
4M7(b5,b9)/b3         # ベース音つき
s4M7(b5,b9)/b3@green  # 色アノテーションは常に末尾
```

### 完全な例
```
:::
---
key: Eb
bpm: 96
---
[サビ]
| NC |
> ぼく
| 1(9) b3@red 2m 37 |
> の記憶 が_すべ て消えて も
| 6m7 3m/5 6m6 4m/5@green |
| 1/3 4 5 37/b6@red |
> _ _ _ 満
| 6m7 3m/5 4m 2-5 |
> 月のよ る_君 をみつけ た
:::
```

## AST 定義

### TypeScript 型定義

```typescript
type ParseResult = {
  score: ScoreAST          // 部分 AST（エラー行は除外される）
  errors: ParseError[]     // 行番号・位置つきの構造化エラー
}

type ScoreAST = {
  meta?: ScoreMeta         // フロントマター（宣言時のみ）
  sections?: SectionLabel[]
  lines: Line[]
}

type ScoreMeta = {
  key?: string             // "C" | "G" | "F#" | "Bb" など
  bpm?: number             // 20-400
  time?: string            // "4/4" など
}

type SectionLabel = {
  line: number             // 元ソースの行番号（1-indexed）
  name: string
}

type Line = {
  line: number
  tokens: Token[]
  lyrics?: (string | null)[]  // スロットに左から割り当てた歌詞断片
}

type Token =
  | { type: 'Chord', chord: ChordNode }
  | { type: 'ChordGroup', chords: ChordNode[] }  // - 連結（1 スロット共有）
  | { type: 'BarLine' }
  | { type: 'Repeat' }                            // %
  | { type: 'NoChord' }                           // NC / N.C.
  | { type: 'EmptyBeat' }                         // _（空の拍）

type ChordNode = {
  accidental?: 's' | 'b'
  degree: '1' | '2' | '3' | '4' | '5' | '6' | '7'
  quality?: 'm' | 'dim' | 'aug'
  seventh?: Seventh
  alteration?: Alteration    // 括弧外のオルタレーション
  options?: ChordOption[]    // 括弧内のオプション
  bass?: BassNote
  color?: 'blue' | 'red' | 'green'
}

type Seventh =
  | { type: 'seventh', value: '7' | 'M7' | '6' }
  | { type: 'add', degree: '2' | '4' | '6' | '9' | '11' | '13' }

type Alteration =
  | { type: 'b5' }
  | { type: 'sus', degree: '2' | '4' | '6' }

type ChordOption = Tension | Alteration

type Tension =
  | { type: 'b9' } | { type: '9' } | { type: 's9' }
  | { type: '11' } | { type: 's11' }
  | { type: 'b13' } | { type: '13' }

type BassNote = {
  accidental?: 's' | 'b'
  degree: '1' | '2' | '3' | '4' | '5' | '6' | '7'
}

type ParseError = {
  kind: string             // "InvalidDegree" など
  message: string
  line?: number            // 1-indexed
  position?: number        // 行内の 0-indexed 文字位置
  token?: string
}
```

### JSON 出力

`parse_to_json` は `{ "success": Bool, "data": ScoreAST, "errors": [...] }` のエンベロープで返す。`data`（ScoreAST）の形状:

```json
{
  "type": "ScoreBlock",
  "meta": { "key": "G", "bpm": 120, "time": "4/4" },
  "sections": [ { "line": 1, "name": "Aメロ" } ],
  "lines": [
    {
      "line": 4,
      "tokens": [
        { "type": "Chord", "chord": { "degree": "1", "seventh": { "type": "seventh", "value": "M7" } } },
        { "type": "BarLine" }
      ],
      "lyrics": [ "ひかりの", null ]
    }
  ]
}
```

## エラーハンドリング

パーサは fail-fast せず、**行単位でエラーを収集**する。エラーのある行は部分 AST から除外され、`errors` 配列に `kind` / `message` / `line` / `position` / `token` を持つ構造化エラーとして記録される。1 行に複数の不正トークンがあれば、その行から複数のエラーが報告される。

| kind | 発生条件 |
|------|---------|
| InvalidDegree | ディグリーが 1-7 以外（ルート欠落 `/3`、不正ベース度数 `1/8` を含む） |
| InvalidSeventh | 不正な add 度数（`1add5` 等） |
| InvalidTension | 括弧内の認識できない要素（`1(b8)` `1(foo)` 等） |
| UnmatchedParenthesis | 閉じ括弧の欠落（`1M7(b9`） |
| EmptyParentheses | 空の括弧（`1M7()`） |
| InvalidCommaUsage | カンマの不正使用（`1(,b9)` `1(b9,)` `1(b9,,s11)`） |
| InvalidColor | 不正な色名・色名欠落・色の後に構成要素（`1@yellow` `1@` `1@red/3` `1@RED`） |
| UnrecognizedToken | 認識できない文字列（`1x`、区切り文字だけのトークン等） |
| DuplicateSeventh | セブンスの重複（`1M77` `17M7` `1M7M7` `1M7add9`） |
| MissingBassDegree | スラッシュ後のディグリー欠落（`1/` `1/b`） |
| InvalidSusDegree | sus 度数が 2/4/6 以外（`1sus5` `1(sus11)` `1(sus)`） |
| UnclosedFrontmatter | フロントマターの閉じ `---` 欠落 |
| InvalidFrontmatterField | 未知のフロントマターフィールド |
| InvalidKey | 不正なキー名（`H` `Gm` `g` 等） |
| InvalidTempo | bpm が整数 20-400 以外 |
| InvalidTimeSignature | 拍子がホワイトリスト外 |
| TooManyLyricFragments | 歌詞断片がスロット数を超過 |
| OrphanLyricLine | 直前がコード行でない歌詞行 |

## 表示仕様

1. **ディグリー表示の変換**: DSL の入力は `1`〜`7` / `b` / `s` のまま変えず、表示層で変換する — ディグリーは `I`〜`VII`（ローマ数字）、`b` → `♭`、`s` → `#`。ベース音・括弧内テンションも同様（`s4M7(b5,b9,s11)/b3` → `#IVM7(♭5,♭9,#11)/♭III`）
2. **コードスロットグリッド**: 各コード・グループ・反復・N.C.・空拍を**スロット**（グリッドの 1 マス）に置き、列数はスコア内の最大スロット数に揃える。行が違っても同じ位置のコード同士が縦に揃う（実譜面のシステム整列のイメージ）。小節線 `|` はスロットを消費せず、**スロット境界の縦線**として描画される。行頭・行末の `|` はそれぞれ開始・終止の縦線になる。列幅は内容に応じて広がる（`minmax(max-content, 1fr)`）ため、長いコードが `...` に切り詰められることはない。`-` グループは 1 スロットを等間隔で分け合う（拍位置の表現）
3. **タブ切り替え**: ブロックは「ディグリー」「コード」の 2 タブを持つ（ブラウザ風タブ: アクティブタブが下のパネルと連結して見える）。**デフォルトタブはディグリー**
4. **コードタブと移調**: キーに基づいてディグリーを実音コード名に変換して表示する（`key: G` で `1 → G`、`3m7 → Bm7`、`5/3 → D/B`）。スペリングはメジャースケールの**レター（C-B）単位**で解決するため異名同音が正しくなる（key F# の `7` は `E#`、key Bb の `b3` は `D♭`）。キープルダウンで**任意のキー**へ移調でき、一覧は 12 半音（`C, D♭, D, E♭, E, F, F#, G, A♭, A, B♭, B`。宣言キーが一覧と異名同音の場合はそのスロットを宣言の綴りで置き換える — `C#` は `D♭` の位置）
5. **エラー表示**: エラー行は元の行位置に行番号・位置つきメッセージ + ソース行として埋め込まれ、正しい行のコード譜は描画され続ける
6. **画像としてコピー**: 「画像コピー」ボタンで表示中のタブの譜面を PNG としてクリップボードへ（不可の環境ではダウンロード）。画像は貼り付け先を選ばないよう常にライトテーマ（白地）で書き出す
7. 色アノテーション・歌詞・セクションは両タブ・全キーで保持される。ウィジェットは上下を区切り線で囲われたブロックとして表示される

## 再生仕様

「▶ 再生」ボタンで、**表示中のキー**（キープルダウンの現在値）の和音を再生する。再生中は鳴っているスロットがハイライトされ、ボタンは「■ 停止」に変わる（再生し切ると自動停止。キー変更で停止）。

拍の解釈は「**1 スロット = 1 小節**」（NNS の「1 ディグリー = 1 小節」の慣習と一致）。小節の長さは拍子（`time`、既定 4/4）で決まる:

| トークン | 音 |
|---------|-----|
| コード | **上声**（ルート・3度・5度 + セブンス/6th/add・テンション。sus は 3度置換、dim/aug/b5 反映）を 1 小節持続。**ベース音**（スラッシュ指定 or ルート）は**拍ごとに刻む**（4/4 なら 4 打） |
| グループ `2-5` | 小節を等分割して順に発音（2 連結なら半小節ずつ。ベースは各コードの頭で単発） |
| `%` | 直前のコードを**再打鍵**（ベースの刻みも再開）。直前のコードが無い行頭の `%` は無音の 1 小節として時間を占有する |
| `_` 空拍 | 直前の発音を 1 小節分**持続**（ベースの刻みは継続。N.C. をまたいだ持続はしない） |
| `N.C.` | 無音の 1 小節 |
| 小節線・セクション・歌詞 | 時間に影響しない |

- テンポは `bpm:`（4 分音符基準。既定 120）
- 拍子とベースの刻み: 4 分系（x/4）は 4 分音符ごと、2 分系（x/2）は 2 分音符ごと、**8 分系の複合拍子（3/8, 6/8, 9/8, 12/8）は付点 4 分のパルス**（6/8 なら 1 小節 2 打）
- ベースの音域は可聴性のため **E2〜E♭3** の窓に収める（低すぎる音はオクターブ上げ。例: key E♭ の `6` は C2 ではなく C3）
- スケジュール生成（ディグリー→MIDI・`%` の解決・拍割り付け）は `parse_to_playback(input, key)` が JSON で返し、クライアントは Web Audio の発音（三角波シンセ）とカーソル表示のみを行う（移調と同じ単一ソース原則）
- パースエラーのある行は描画と同様に再生からも除外される

## 予約トークン（将来拡張）

以下の小節線バリエーションは ChordPro grid 語彙を正として**将来のために予約**する。現行仕様での動作: `||` は 2 つの小節線、`%%` は 2 つの反復記号として解釈される。将来のバージョンで意味が変わる可能性があるため、使用しないこと。

`||`（複縦線）、`|.`（終止線）、`|:` `:|` `:|:`（リピート）、`|1` `:|2`（ボルタ）、`%%`（2小節リピート）

## 設計判断メモ

- **`-` グループ採用によるトレードオフ**: ジャズ記法のマイナー略記（`6-`）は将来もサポートしない（`2-5` と曖昧になるため）。マイナーは常に `m`
- **`_` の三義**: コード行 = 空拍（スロット占有）/ 歌詞行の単独 `_` = スキップ / 歌詞断片中の `_` = スペース
- **色は 3 色固定**: 記事内の視覚言語を単純に保つ（筆者が記事ごとに「赤 = 主題」等の意味づけをする余地を残す）。色価はテーマ側の CSS で決まる
- **マイナーキー**（`key: Am` 等）は未サポート。NNS の慣習では相対長調で記譜する
- **キー・変換・再生の音楽計算は MoonBit 側の単一ソース**。クライアント（JS）は表示と発音のみを担い、ロジックを二重実装しない
